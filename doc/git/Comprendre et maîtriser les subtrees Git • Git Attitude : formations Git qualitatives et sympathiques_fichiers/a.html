                          Christophe Porteneuve Mod ouaibsky • 10 months ago

                          Alors tout dépend si "les autres" doivent aussi faire des manips de synchro sur ces subtrees (récup de leur upstream, backport vers leur upstream) ou non.

                          Dans le cas où ils ne font pas de telles opérations de maintenance, tu n'as **rien de spécial à faire** : ton dépôt principal reste un seul et unique dépôt.

                          Dans le cas où il y a en effet ce besoin, ils peuvent pour le moment faire le même git stree add que toi, une seule fois, après avoir cloné le dépôt (ou en cours d'utilisation de ce dépôt). À terme, le .gitstrees automatisera ça, en effet.
